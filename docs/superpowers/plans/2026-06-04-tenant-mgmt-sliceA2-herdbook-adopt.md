# Tenant Management — Slice A2 (Herdbook Adopt) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`). Work in a git worktree off `domains/herdbook` `main`.

**Goal:** Flip herdbook onto the substrate's persisted-tenant registry (`PrismaTenantRegistry`, `@de-braighter/substrate-runtime@0.20.0`) and canonicalize its `tenant_pack_id` from the string stub `herdbook-tpid-0001` to the derived UUID `28b96bdb-2321-5a5e-9b66-3b49af0a5940`, so all three subsystems (pack data, the RLS GUC, and the policy grant) agree on one value.

**Architecture:** A **re-seed flip**, not a data-UPDATE migration. herdbook is pre-launch; `db:setup` regenerates all data, so changing the `HERDBOOK_TENANT_PACK_ID` constant + seeding under the UUID is the clean path (avoids the RLS `USING`/`WITH CHECK` GUC conflict an in-place re-key would hit). `db:setup` gains two steps: provision the herdbook tenant (`core.tenant` + `core.tenant_pack` via the tested `provisionTenant` helper, which derives the UUID) and seed the registrar grant under the UUID. `app.module` (via `dbBackedAuthForRootOptions`) binds `tenantRegistry: new PrismaTenantRegistry(appRoleClient)` on the DB-auth path; the in-memory/dev path keeps `tenants: HERDBOOK_TENANTS`. The 140 test usages auto-follow the changed constant.

**Tech Stack:** TypeScript ESM, NestJS, Prisma 6 (hand-written migrations, `migrate deploy`), pnpm workspace, Vitest (`describe.skipIf` DB-gated), the herdbook `tools/db/setup.mjs` harness, Postgres 16 RLS on :5433.

**Spec:** umbrella `docs/superpowers/specs/2026-06-03-tenant-management-design.md` §3.3 + Slice A; ADR-209 (ratified). **Predecessor:** Slice A1 shipped `substrate-runtime@0.20.0`.

## Task 4: Provision the herdbook tenant + registrar grant in db:setup

**Files:** Create `libs/herdbook-pack/prisma/seed/provision-tenant.ts` + `registrar-grant.ts`; modify `tools/db/setup.mjs`.

- [ ] **Step 1: Write `provision-tenant.ts`.** Runs as the owner (writes the auth-scoped core tables). Reuses the tested helper.
```ts
import { PrismaClient } from '@prisma/client';
import { provisionTenant } from '@de-braighter/substrate-runtime';

const HERDBOOK_TENANT_ID = 'b6c5d8e2-0001-4abc-9def-fedcba000001';

async function main(): Promise<void> {
  const url = process.env['DATABASE_URL']; // setup.mjs passes the owner/migrate URL
  if (!url) throw new Error('provision-tenant: DATABASE_URL required');
  const db = new PrismaClient({ datasources: { db: { url } } });
  try {
    await provisionTenant(db, {
      tenantId: HERDBOOK_TENANT_ID,
      slug: 'herdbook',
      displayName: 'Breed registry association',
      packKeys: ['herdbook'],
    });
    console.log('[seed] provisioned core.tenant + core.tenant_pack for herdbook');
  } finally {
    await db.$disconnect();
  }
}
main().catch((e) => { console.error(e); process.exit(1); });
```
- [ ] **Step 2: Write `registrar-grant.ts`.** Seeds the demo registrar grant under the UUID. `core.pack_role_assignment` is FORCE RLS → set the GUC = the UUID, then insert (WITH CHECK passes since the row's tenant_pack_id = GUC).
```ts
import { PrismaClient } from '@prisma/client';

const TPID = process.env['HERDBOOK_TENANT_PACK_ID'] ?? '28b96bdb-2321-5a5e-9b66-3b49af0a5940';
const DEV_USER = '11111111-1111-4111-8111-111111111111';

async function main(): Promise<void> {
  const url = process.env['DATABASE_URL'];
  if (!url) throw new Error('registrar-grant: DATABASE_URL required');
  const db = new PrismaClient({ datasources: { db: { url } } });
  try {
    await db.$transaction(async (tx) => {
      await tx.$executeRawUnsafe(`SELECT set_config('app.tenant_pack_id', $1, true)`, TPID);
      await tx.$executeRawUnsafe(
        `INSERT INTO "core"."pack_role_assignment" ("tenant_pack_id","user_id","pack_key","role_id")
           VALUES ($1::uuid, $2, 'herdbook', 'registrar')
           ON CONFLICT DO NOTHING`,
        TPID, DEV_USER,
      );
    });
    console.log('[seed] registrar grant seeded for the dev user under', TPID);
  } finally {
    await db.$disconnect();
  }
}
main().catch((e) => { console.error(e); process.exit(1); });
```
